## 運用ルール

- 新しいサービス案やプロトタイプを作ったら、この一覧に追加する。
- 企画、調査、プロトタイプ、実装 repo が分かれる場合は、最低1つの README から辿れるようにする。
- GitHub で管理する実装 repo がある場合は、各 project README に remote URL とローカル実体パスを書く。
- PR / branch / merge の共通ルールは `company/engineering/docs/2026-06-24-github-pr-policy.md` に従う。
