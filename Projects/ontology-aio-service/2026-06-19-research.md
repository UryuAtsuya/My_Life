# AIO Service Research 2026-06-19

## 要約

AIO / GEO / AEO 領域は、2026年時点で「AI 検索にどう表示されるかを測る monitoring ツール」と「SEO 会社・マーケ支援会社の新メニュー」に分かれて広がっている。

Google 公式は、AI Overviews / AI Mode 向けの最適化を特別な別分野としてではなく、基本的には SEO の延長として説明している。つまり「AIO 対策」として売る場合でも、露出保証や裏技ではなく、クロール可能性、信頼できる一次情報、独自性、構造化された公式情報、外部からの自然な言及を整える支援として設計する方が安全。

既存競合は、ブランドが ChatGPT、Perplexity、Google AI Overviews / AI Mode、Copilot、Gemini などでどれだけ言及・引用・推薦されるかを測る機能に集中している。差別化するなら、測定 SaaS そのものよりも、測定結果をもとに Entity Map、Content Graph、schema.org、FAQ、比較ページ、一次情報ページまで設計する「実装可能なブランド理解設計」に寄せる。

## 市場の見え方

- 呼称は `AIO`, `AEO`, `GEO`, `AI visibility`, `LLM visibility`, `AI search monitoring` が混在している。
- Google は `AEO` / `GEO` を Google Search 目線では「still SEO」と扱っている。
- AI 検索の対象は Google だけではなく、ChatGPT、Perplexity、Gemini、Claude、Copilot、Grok、DeepSeek などに広がっている。
- 顧客が知りたいのは「AI に表示されるか」だけではなく、「どの競合と比較されるか」「どの情報源が引用されるか」「ネガティブに説明されていないか」「改善施策に落とせるか」。
- 低価格帯は prompt tracking、citation tracking、brand mention tracking が中心。高価格帯は enterprise reporting、crawler / bot analytics、traffic attribution、content workflow まで広がる。

## 競合カテゴリ

### 1. AI Visibility Monitoring

代表例:

- Profound
- Peec AI
- Otterly.AI
- Semrush AI Visibility Toolkit
- BrightEdge / Conductor 系の enterprise SEO suite

主な機能:

- AI answer 内の brand mention tracking
- share of voice / visibility score
- competitor comparison
- citation source / cited URL tracking
- sentiment analysis
- prompt library / prompt discovery
- daily tracking
- region / language tracking
- AI crawler / bot access analysis
- reporting / export / API / Looker Studio 連携

弱点:

- 測定と可視化は強いが、顧客側が何を直すべきかの実装設計は別途必要になりやすい。
- SEO / content チームが強くない顧客には、dashboard だけ渡しても動けない。
- ツールごとの測定方法が揺れるため、スコアそのものを絶対値として売るのは危険。

### 2. SEO Suite の AI 機能

代表例:

- Semrush AI Visibility Toolkit / Enterprise AIO
- BrightEdge
- Conductor
- Ahrefs Brand Radar 系

主な機能:

- 既存 SEO データと AI visibility の統合
- prompt research
- competitor gap
- technical audit
- report generation
- enterprise 向け workflow

弱点:

- 既存 SEO ツールの延長に見えやすく、Ontology / Knowledge Graph 設計の専門性は前面に出にくい。
- SMB や専門サービス事業者には重い可能性がある。

### 3. コンサル / 代理店メニュー

代表例:

- SEO 会社の AIO / GEO 支援メニュー
- PR / reputation management 会社の AI brand monitoring
- Web 制作会社の structured data / content 改善メニュー

主な機能:

- 監査レポート
- コンテンツ改善案
- FAQ / 比較記事 / 用語集制作
- schema.org 実装
- 外部言及 / PR / review 獲得支援

弱点:

- 「AI 検索最適化」と言いながら、実態が従来 SEO 記事制作に寄りがち。
- AI answer の観測設計、Entity / Ontology 設計、実装指示まで一貫して出せる会社はまだ差別化余地がある。

## 主要ツールメモ

### Profound

Enterprise 寄りの AI visibility platform。AI Visibility、Source Citations、Brand Sentiment、Content AEO などを前面に出している。対象 engine は Perplexity、ChatGPT、Claude、Gemini、Grok、Microsoft Copilot、Meta AI、DeepSeek、Google AI Overviews など。Agent Analytics により AI bot がサイトをどうクロールしているかも扱う。

示唆:

- 大企業向けは `visibility + citations + sentiment + crawler analytics + reporting` が標準機能化している。
- 自社サービスで正面から dashboard SaaS として戦うより、診断結果を解釈して実装設計に落とす方が差別化しやすい。

### Peec AI

Marketing team / agency 向けの AI search analytics。brand visibility、sentiment、share of voice、average position、crawl insights などを扱う。server log 由来の AI bot activity を「first-party data」として重視している。

示唆:

- 代理店・複数ブランド運用向けの需要がある。
- prompt / topic / tag で切り分ける運用が重要。単なる URL 診断より、顧客の事業カテゴリ・商談クエリ・競合軸ごとの設計が必要。

### Otterly.AI

比較的入りやすい価格帯の AI search monitoring。ChatGPT、Google AI Overviews、Perplexity、Microsoft Copilot などを tracking し、brand mention、citation、competitor visibility を見る。pricing は prompt 数に連動しており、Lite / Standard / Premium のように段階化されている。

示唆:

- 低価格 monitoring は既に存在するため、`URL を入れるだけの簡易スコア SaaS` は競争が激しい。
- ただし、低価格ツールのデータを使ってレポート作成・改善指示を請け負う partner / consultant 需要はあり得る。

### Semrush AI Visibility Toolkit / Enterprise AIO

既存 SEO 顧客向けに、AI-generated answers 上の brand visibility、mentions、sentiment、competitor research、prompt research、daily prompt tracking、technical crawler issue audit などを提供。Enterprise AIO は大規模 prompt database、traffic logs、authority signals、GA4 / Adobe Analytics 連携を押し出している。

示唆:

- 既存 SEO suite はデータ量と顧客基盤が強い。
- 小規模に始めるなら、Semrush などと同じ土俵のデータ量勝負は避け、`何を直すか` の設計品質で勝つべき。

## Google 公式ガイドからの制約

Google Search Central の 2026年6月時点のガイドでは、生成 AI 検索でも foundational SEO が引き続き重要とされている。

重要点:

- AI Overviews / AI Mode は Google Search の ranking / quality systems と関係している。
- eligible になるには、ページが index され、snippet 表示可能で、技術要件を満たす必要がある。
- crawlable な公開 content が重要。
- unique, valuable, people-first content が長期的に重要。
- Merchant Center や Google Business Profile は product / local business の visibility に関係する。
- `llms.txt`、AI 専用 markup、AI 専用 rewriting、content chunking、特殊 schema は Google Search 向けには不要と説明されている。
- structured data は Google の generative AI search に必須ではないが、通常 SEO / rich results の文脈では引き続き有用。

サービス設計上の注意:

- 「llms.txt を入れれば AI Overview に出る」のような売り方はしない。
- 「特殊 schema で AIO を攻略する」のような表現も避ける。
- 売るべき価値は `AI が参照しやすい公式情報基盤` と `人間にも検索エンジンにも有用な情報構造`。

## Ontology AIO Service の勝ち筋

### 避けるべき方向

- 汎用 AI visibility dashboard SaaS
- prompt 数・engine 数・スコア精度だけの競争
- SEO 記事量産代行
- AI Overview 露出保証
- llms.txt / 特殊 schema だけを売る technical hack

### 寄せるべき方向

- `AI Search Brand Understanding Audit`
- `Entity / Ontology 設計レポート`
- `Content Graph 実装指示`
- `公式情報・比較情報・FAQ・一次情報ページの設計`
- `schema.org / JSON-LD / internal linking / author / organization 情報の実装指示`
- `AI answer simulation + 改善 backlog`
- SEO 会社・Web 制作会社向けの裏方レポート生成

### 差別化コピー案

AI 検索で「表示されるか」だけでなく、AI に「どう理解されるか」を設計する。

### 商品名案

- AI Search Brand Understanding Audit
- Entity AIO Audit
- LLM Brand Map
- AI Visibility Implementation Plan
- Ontology SEO / AIO Design

## 最初の検証案

### 1. 手動レポートを1件作る

対象は B2B SaaS、士業、クリニック、採用支援、専門 EC のいずれか。

成果物:

- 主要商談 prompt 10個
- AI answer simulation
- 競合 mention / citation / sentiment
- Entity Map
- Content Gap
- 推奨 Content Graph
- schema.org 改善案
- 30日実装 backlog

### 2. 診断項目をスコア化する

初期スコア案:

- Entity clarity
- Category ownership
- Competitor differentiation
- Evidence / proof richness
- Official source completeness
- Citation readiness
- Structured data hygiene
- External mention quality
- AI answer risk

### 3. 既存ツールを使うか自前で測るかを分ける

自前で作る:

- Entity extraction
- Content Graph
- schema.org suggestion
- report template
- implementation backlog

既存ツール利用でよい:

- prompt monitoring
- daily tracking
- multi-engine answer collection
- citation tracking
- competitor visibility trend

### 4. 価格仮説

- Entry: 簡易診断 0円から3万円
- Core: Entity / Content Graph 設計 10万から30万円
- Premium: 月次改善 15万から50万円
- Partner: SEO 会社向け月額 10万から。テンプレート、監査、実装指示、提案資料を提供

## 次の一手

1. サンプル対象サイトを1つ決める。
2. `AIO診断レポートテンプレート.md` を作る。
3. 手動で10 promptだけ実行し、AI answer / citation / competitor mention を記録する。
4. Entity Map と Content Gap を作る。
5. 30日実装 backlog に落とし、営業用の1枚提案に変換する。
