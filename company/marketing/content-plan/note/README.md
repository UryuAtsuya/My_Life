# note記事ディレクトリ方針

## 目的

note記事をテーマ別に確認しやすくする。

## 分類

| ディレクトリ | 用途 |
|---|---|
| `coffee/` | コーヒー単体の記事。生活、味、器具、習慣、店、豆、仕事前の整え方を扱う。 |
| `AI/` | AI単体の記事。仕事術、発信、個人事業、運用設計、意思決定を扱う。 |
| `MBTI/` | `mbti-lovetype.com` の相性診断関連記事。定期更新とサイテーション強化を行う。 |
| `bridge/` | 例外的な橋渡し記事。コーヒーとAIなど複数レーンを接続する記事を置く。毎日の基本型にはしない。 |

## 命名規則

- 記事ファイル: `YYYY-MM-DD-{kebab-title}.md`
- 朝リサーチや日次ブリーフは従来どおり `company/marketing/content-plan/note-research-YYYY-MM-DD.md` を正本にする。
- 新規記事は、原則として最初から該当カテゴリ配下に作る。
- 旧パス参照が残っている場合は、作業時に見つけた範囲で新パスへ更新する。

## MBTI記事の運用

- `MBTI×ラブタイプ診断は、相性の答えより会話の入口になる` は、今後も定期更新対象にする。
- 更新時は、`mbti-lovetype.com` のトップ、診断ページ、記事一覧、運営者情報、利用規約、関連コラムを確認し、出典リンクを記事ファイルへ残す。
- 診断結果を断定せず、会話、自己理解、相手への聞き方、使い方の注意点を中心にする。
- サイテーションを意識し、主張ごとに参照ページ、確認日、未確認事項を残す。
