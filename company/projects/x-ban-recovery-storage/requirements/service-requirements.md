# XGuard 要件定義

## サービス概要

XでBANリスクのあるクリエイター向けに、投稿・フォロワーデータを毎日自動バックアップし、BAN発生時に本人証明ページの発行とフォロワーへの移行通知を行うSaaSサービス。

## 価格

- 月額: 3,000円
- 課金方式: Stripeサブスクリプション
- 初期案: 単一プラン

## ターゲット

- X依存度が高いクリエイター
- 収益導線、信用、フォロワー接点をXに依存している個人事業者
- BANや凍結後に「同一人物証明」と「新アカウント告知」を急ぐ必要がある人

## 提供価値

1. 平常時の投稿・プロフィール・フォロワーデータを毎日保存する。
2. BAN候補を検知し、ユーザーへ通知する。
3. 保存済みデータから本人証明ページを発行する。
4. 新アカウントへの移行告知文を生成する。
5. フォロワーへの移行通知をキュー化する。

## 重要な表現ルール

- 「BANされたアカウントを復活させる」とは言わない。
- 「BAN後に新アカウントで再起動するための証明・移行支援」と表現する。
- 自動DMや一括フォローは、X API規約とスパム判定リスクを確認してから実装判断する。

## MVP機能

| 機能 | 内容 | 優先度 |
|---|---|---|
| 認証 | Supabase Authでユーザー登録・ログイン | P0 |
| サブスク課金 | Stripeで月額3,000円の購読管理 | P0 |
| X接続 | X API連携用の認証情報登録 | P0 |
| 毎日バックアップ | 投稿、プロフィール、フォロワー関連データを保存 | P0 |
| BAN候補検知 | X APIの403/404/認証失敗などを分類して通知 | P0 |
| 証明ページ | 保存済み投稿、プロフィール、活動実績を公開ページ化 | P0 |
| 移行文生成 | 新アカウント告知文、固定投稿文、プロフィール文を生成 | P1 |
| フォロワー通知キュー | 通知候補をキュー化。自動送信は後回し | P2 |

## 成功条件

- ユーザーがXアカウントを接続できる。
- 毎日バックアップがDBに保存される。
- BAN候補検知の結果がユーザーに通知される。
- 新アカウント告知に使える証明ページURLが発行される。
- 月額3,000円の購読状態で機能利用可否を制御できる。

## 未確定事項

- X APIでフォロワーリスト、フォロー中リスト、DM送信をどこまで扱えるか。
- X API料金と月額3,000円の原価バランス。
- 証明ページに表示してよい第三者情報の範囲。
- BAN候補検知で403/404をどう分類するか。
