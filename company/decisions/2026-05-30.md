# Decisions - 2026-05-30

## Morning XGuard research meeting

1. 今日の事業最優先はXGuardにする。note/Today Board/その他webサービス改善は、実Supabase migration test、real OAuth configured mode、Developer Console原価確認が進むまで後回しにする。
2. XGuard v0は、引き続き「BAN復活」ではなく「本人が事前同意したXデータのread-only backup」と「BAN後の手動再起動を支えるproof page」に限定する。
3. v0初期OAuth scopeは `tweet.read`, `users.read`, `offline.access` に維持する。`follows.read`, `dm.read`, `dm.write`, `tweet.write`, `follows.write` は初期同意画面に入れない。
4. `record_api_usage_event_with_monthly_limit` は、実Supabase/Postgres上でも `service_role` 専用実行、所有関係検証、同一Xアカウント整合性、月次上限超過、負値拒否を確認するまで完了扱いにしない。
5. X API原価はDeveloper Console実画面確認を正とする。確認できるまで、通常read単価と月次原価上限を保守的に使い、`Owned Reads` を第三者ユーザー向けSaaSの主前提にしない。
6. proof pageはraw X API payloadを公開せず、公開DTO、visibility、revocation、redaction、X Content削除/非公開/withheld追従を実装境界に入れる。
7. 自動DM、自動follow/unfollow、自動投稿、bulk outreach、BAN回避に見える新アカウント自動作成・誘導はv0から外す。
8. 昼の最初のゲートは `/Users/uryuatsuya/XGuard/xguard` のwrite/fetch/remote最新確認にする。朝run時点では `exists=yes`, `writable=no`, `main...origin/main`, HEAD `d30fc48`。
