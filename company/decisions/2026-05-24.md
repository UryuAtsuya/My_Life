# Decisions - 2026-05-24

## Morning XGuard research meeting

1. 今日の事業最優先はXGuardにする。note/Today Board/その他webサービス改善は、XGuardのDB schema v1とread-only実装境界が固まるまで後回しにする。
2. XGuard v0は「BAN復活」ではなく、「本人が事前に許可したXデータのread-onlyバックアップ」と「BAN後の再起動支援用の証明ページ生成」に限定する。
3. v0のP0 endpointは `GET /2/users/me`, `GET /2/users/:id`, `GET /2/users/:id/tweets` にする。followers/following個別一覧はP1で、昼は公開しない前提の設計に留める。
4. `tweet.write`, `follows.write`, `dm.write`、自動DM、自動follow/unfollow、自動投稿、bulk outreachはv0から外す。
5. 証明ページはraw X API payloadを公開しない。`proof_pages.public_payload` のような公開DTOを分離し、visibility、revocation、redaction policyを持たせる。
6. X Contentの削除・変更・protected化・withheld・ユーザー削除要求に追従する `content_compliance_events` をDB schema v1へ入れる。
7. X APIはPay-per-useとOwned Reads前提で原価管理する。昼は `api_usage_events`, `backup_runs`, `docs/API_COST_MODEL.md` を作り、Developer Console確認待ちを明示する。
8. 昼の実装対象は `/Users/uryuatsuya/XGuard/xguard`。MyLife Vaultには実装コードを置かず、会社側には計画・判断・PMメモだけを残す。
