# Decisions - 2026-05-27

## Morning XGuard research meeting

1. 今日の事業最優先はXGuardにする。note/Today Board/その他webサービス改善は、指定パス同期、ledger validation、Developer Console原価確認が進むまで後回しにする。
2. XGuard v0は、引き続き「BAN復活」ではなく「本人が事前同意したXデータのread-onlyバックアップ」と「BAN後の手動再起動を支える証明ページ」に限定する。
3. v0初期OAuth scopeは `tweet.read`, `users.read`, `offline.access` に維持する。`follows.read`, `dm.read`, `dm.write`, `tweet.write`, `follows.write` は初期同意画面に入れない。
4. X API原価はDeveloper Console実画面確認を正とする。確認できるまで、Post read `$0.005/resource`、User/Followers read `$0.010/resource` の保守的値で見る。
5. `Owned Reads` の `$0.001/resource` は、第三者ユーザー向けSaaSの通常ユーザーに適用できると確認できるまで主前提にしない。
6. proof pageはraw X API payloadを公開せず、公開DTO、visibility、revocation、redaction、X Content削除/非公開/withheld追従を実装境界に入れる。
7. 自動DM、自動follow/unfollow、自動投稿、bulk outreach、BAN回避に見える新アカウント自動作成・誘導はv0から外す。
8. 昼の最初のゲートは `/Users/uryuatsuya/XGuard/xguard` の書き込み可否と `origin/main` `c7a315c` 以降への同期にする。朝run時点では `exists=yes`, `writable=no`, `main...origin/main [ahead 4, behind 2]`。
9. 昼の実装対象は、`ApiUsageLedgerService` の非負整数validationと失敗テスト、Developer Console実値確認、`docs/API_COST_MODEL.md` 更新の順にする。
