# Decisions - 2026-05-26

## Morning XGuard research meeting

1. 今日の事業最優先はXGuardにする。note/Today Board/その他webサービス改善は、XGuardのremote同期、TypeScript検証復旧、X API原価確認が進むまで後回しにする。
2. XGuard v0は、引き続き「BAN復活」ではなく「本人が事前同意したXデータのread-onlyバックアップ」と「BAN後の手動再起動を支える証明ページ」に限定する。
3. v0初期OAuth scopeは `tweet.read`, `users.read`, `offline.access` に維持する。`follows.read`, `dm.read`, `tweet.write`, `follows.write`, `dm.write` は初期同意画面に入れない。
4. `Owned Reads` の `$0.001/resource` は、第三者ユーザー向けSaaSに適用できると確認できるまで主前提にしない。Post read `$0.005/resource`、User/Followers read `$0.010/resource` で保守的に原価を見る。
5. proof pageはraw X API payloadを公開せず、公開DTO、visibility、revocation、redaction、X Content削除/非公開/withheld追従を実装境界に入れる。
6. 自動DM、自動follow/unfollow、自動投稿、bulk outreach、BAN回避に見える新アカウント自動作成・誘導はv0から外す。
7. 昼の最初のゲートは `/Users/uryuatsuya/XGuard/xguard` の書き込み可否とremote統合にする。朝run時点では `exists=yes`, `writable=no`, `main...origin/main [ahead 2]`。
8. 昼の実装対象は、remote `b3bd37c` とlocal `f60be3e`/`91229db` の統合、`tokenRepository.test.ts` 型修正、`findXToken()` scope再検査、usage/cost ledger最小contractの順にする。
