# Decisions - 2026-05-25

## Morning XGuard research meeting

1. 今日の事業最優先はXGuardにする。note/Today Board/その他webサービス改善は、XGuardのdocs gate、build gate、token repositoryの最小実装が進むまで後回しにする。
2. XGuard v0は「BAN復活」ではなく、「本人が事前に許可したXデータのread-onlyバックアップ」と「BAN後の手動再起動を支援する証明ページ生成」に限定する。
3. v0初期OAuth scopeは `tweet.read`, `users.read`, `offline.access` に維持する。`follows.read`, `dm.read`, `tweet.write`, `follows.write`, `dm.write` は初期同意画面に入れない。
4. X API原価はPay-per-usageとDeveloper Console実画面確認を正とする。`Owned Reads` の低単価枠を第三者ユーザー向けSaaSの主前提にしない。
5. proof pageはraw X API payloadを公開しない。公開DTO、visibility、revocation、redaction、削除追従を実装境界に入れる。
6. 自動DM、自動follow/unfollow、自動投稿、bulk outreach、BAN回避に見える新アカウント自動作成・誘導はv0から外す。
7. 昼の最初のゲートは `/Users/uryuatsuya/XGuard/xguard` の書き込み可否確認にする。朝run時点ではディレクトリは存在し、git statusはcleanだが、現在のCodexサンドボックスからは `NOT_WRITABLE`。
8. 昼の実装対象は `docs/ARCHITECTURE.md`, `docs/API_COST_MODEL.md`, `docs/COMPLIANCE.md`, build設定分離、Supabase token repository最小実装の順にする。
