**状況:**
- `RUN_SUPABASE_SQL_INTEGRATION_TESTS=1` 実行は DB URL / `psql` 条件不足で 2026-05-28 以降毎日 skip 継続。

- **期限**: 2026-06-10 までに環境確保を目標にする。
- それまでは integration test skip を許容し、他の release gate を優先する。

## 判断 3: OAuth Top 1（実 X token exchange）の優先順位

**状況:**
- live X credentials（`X_CLIENT_ID` / `X_CLIENT_SECRET` / production callback URL）がないと sandbox では検証不可。
- P0 指定で毎日繰り越している。

**判断:**
- **production release No-Go** を維持する。
- live credentials を持っている場合は、手動で `/api/x/oauth/callback` の実 token exchange を一度だけ通し、subject/account 検証が機能することを確認する。
- sandbox では `7e33e9f` の boundary gate（mock callback 禁止、production 起動時の configured mode 強制）を確認し、live 検証は別途行う。
- runtime gate 実装（`PRICING_CONFIRMED` / `COMPLIANCE_CONFIRMED`）を先に完了させ、Top 1 live 検証はその後のフェーズとする。
