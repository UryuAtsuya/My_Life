# webサービス新規制作の朝昼夜ループを開始する

## 内容

note運用とは別プロジェクトとして、新しいwebサービス制作を始める。日次の流れは、朝企画、昼コーディング、夜フィードバックに分ける。

## 完了条件

- [ ] 朝企画で、最初に作るwebサービス案を1つに絞る
- [ ] 対象ユーザー、解決する痛み、最小機能、初回画面を決める
- [ ] 昼コーディングで、`Projects/` 配下に最小プロトタイプまたは実装計画を作る
- [ ] 夜フィードバックで、動作確認、未完了、翌日の改善TODOを記録する

## 判断ルール

- 最初から大きく作らない。1日で確認できる最小機能に絞る。
- 実装前に、既存の `Projects/` 候補を確認する。
- note記事制作と混ぜず、webサービス制作は別レーンとして扱う。
- 制作ログは後日AI記事の一次情報として再利用できるように残す。
