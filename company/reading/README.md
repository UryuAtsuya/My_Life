# Weekly Reading

朝のautomationが作る、仕事と息抜きのための短い読み物を保存する。

| ディレクトリ | 内容 |
|---|---|
| `it-trends/` | IT・AI・開発ツールの最新動向 |
| `coffee/` | ハンドドリップ、豆、器具、抽出の読み物 |

## 運用ルール

- 月曜朝に各カテゴリ1本まで作る。
- 1本は600〜1,000字程度に抑える。
- IT記事は原則として直近7日間の一次情報を優先する。
- コーヒー記事は流行だけでなく、試せる抽出方法や味の違いを優先する。
- タイトルだけ変えた重複記事は作らない。
- note公開用の原稿ではなく、個人用の読み物として書く。
- ファイル名は `YYYY-MM-DD-topic.md` とする。
